Notes on Hoare Logic

1. Introduction

Hoare Logic is a logic for progamming languages. It is a logic to verify how programs are calculating. It is a logic to express what those programs are doing. It is a logic to prove programs.

For example, we can imagine a C program which adding two integers. 

int add(int x, int y) {
    int z = 0;
    z = x + y;
    return z;
}

Then, add(2,3) returns $5$, add(-100, 100) returnds $0$.

To verify, express, prove the program, Hoare Logic introduce Hoare triple, axioms and the rules.

Hoare triple is a (P, S, Q) that is denoted: \[\{P\}\, S\, \{Q\}\] where $P$ is the precondition that is true before $S$ is executed, $S$ is a subset of program list, Q is the postcondition that is true after the $S$ are executed.

For example,

$\{x + y \Leftrightarrow 4\}$

z = x + y;

$\{z \Leftrightarrow 4\}$

Here, the precondition $P$ is $x + y \Leftrightarrow 4$, and a program $S$ is z = x + y, and the postcondition $Q$ is $z \Leftrightarrow 4$.

What we want to do with Hoare Toriple is to prove $Q$ from the $P$ and $S$. To do this, we need an axiom of programs.

\begin{equation} \{Q[\text{f/x}]\}\,\,\text{x = f}\,\, \{Q\} \end{equation} where x is a variable of a program, f is a expression of a program, and $Q[\text{f/x}]$ denotes a replacement all occurense of f in $Q$ with x .

Having Axiom of Assignment, now we can prove the following Hoare Triple.

\begin{equation*} \{x + y \Leftrightarrow 4\} \, \text{z = x + y;} \, \{z \Leftrightarrow 4\} \end{equation*}

$z \Leftrightarrow 4$ can be replaced with $(z \Leftrightarrow 4)[\text{x+y/z}]$ by Axiom of Assignment, then we immediately get $ x + y \Leftrightarrow 4$.

$\Box$

This is a super simple toy example, but likewise, we can define several Axioms to prove programs.
